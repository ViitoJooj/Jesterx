# Jesterx

Jesterx 是一个**页面创建 SaaS 平台**。  
初始版本（v1）将在 GitHub 上开放和公开，既作为功能性产品，也作为技术实验。该项目专注于**代码简洁性**、实践学习和想法验证。  
它的存在是为了测试我的 **Golang** 技能，研究可扩展架构，并有可能在未来产生收益。

### 技术栈和技术决策
- 后端使用 **Golang + Gin**
- 前端使用 **React + TypeScript**
- 用户数据使用 **PostgreSQL**
- 网站持久化使用 **MongoDB**
- 缓存和可扩展性使用 **Redis**
- 通知和异步任务使用 **RabbitMQ**
- 通过 **OAuth2** 进行身份验证
- 公共 API
- 全部使用 **Docker 容器化**
- 基础设施部署在 **Azure**，配备 **CI/CD**
- **使用 Python 进行自动化测试**，选择它是因为简单快速

---

## 概述
此仓库汇集了启动电子商务所需的一切：
- 业务规则 API
- Web 界面
- 数据库结构

其目的是让任何开发者都能克隆项目，启动环境，并在无需太多初始配置的情况下开始工作。

---

## 后端
后端集中了所有应用程序逻辑，例如：
- 用户身份验证
- 产品
- 订单
- 数据库通信

API 遵循 REST 模式，如有需要可在未来进行调整。

---

## 前端
前端负责商店界面，包括：
- 产品列表
- 购物车
- 登录和注册
- 结账
- 管理区域（管理员仪表板）

它直接使用后端 API。

---

## 管理区域
- 只有在 `ADMIN_EMAILS` 中列出电子邮件的用户才能获得 `platform_admin` 角色。
- 仪表板显示已创建用户、平均订单金额、收入和最常用计划的指标。
- 用户管理（查看、编辑个人资料和计划数据而不暴露密码、禁止/解禁、删除）。
- 计划管理（更改价格、描述、限制和优惠）。结账始终使用更新后的值。
- 直接从界面和通过 API 导出 XLSX 格式的用户数据。

---

## 数据库
`migrations/` 文件夹中包含数据库脚本，包括：
- 创建表
- 关系
- 初始数据（如适用）

## 环境配置

### 环境变量
复制示例文件：
```bash
cp .env.example .env
```
然后根据您的环境调整变量，例如数据库、端口和访问密钥。

在 `ADMIN_EMAILS` 中包含管理员电子邮件列表（用逗号分隔），以启用 `/admin` 仪表板和 `/v1/admin` 路由。

## Docker
该项目有一个 _docker-compose.yml_ 文件以便于本地设置：
```bash
docker compose up -d
```
这将启动后端、前端、redis、rabbitMQ 和数据库。
